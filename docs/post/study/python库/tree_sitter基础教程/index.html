Tree_sitter基础教程

时间： 2022-04-19   |   分类： 学习   | 字数： 1324 字 | 阅读： 3分钟 | 阅读次数：

安装

首先需要安装VC++14.0或以上。下载链接：https://docs.microsoft.com/en-US/cpp/windows/latest-supported-vc-redist?view=msvc-170

tree sitter 基本使用方法

  1. 使用languages.so 编译成python可以使用的库
from tree_sitter import Language, Parser

Language.build_library(
  # Store the library in the `build` directory
  'build/my-languages.so',

  # Include one or more languages
  [
    'vendor/tree-sitter-go', # tree-sitter对应不同语言的实现。每个需要解析的语言都需要提供。
    'vendor/tree-sitter-javascript',
    'vendor/tree-sitter-python'
  ]
)

# Load the languages into your app as Language objects:
PY_LANGUAGE = Language('build/my-languages.so', 'python')
  1. 基础使用方法
    1. 申明一个Parser
    # lang in python, ruby, java, php, go, javascript 等等
lang = "python"
LANGUAGE = Language('parser/my-languages.so', lang)
parser = Parser()
parser.set_language(LANGUAGE)
    1. 使用parser去解析代码（要求是二进制的代码串）
    tree = parser.parse(bytes("""
def foo():
    if bar:
        baz()
""", "utf8"))
    1. tree即整颗AST树，可以获取根节点之后开始遍历，也可以使用tree的某些方法
    # 获取根节点
root_node = tree.root_node

基础API

Tree 结构体

  1. 所有公开方法：
dir(tree)
>>> ['__class__', '__delattr__', '__dir__', '__doc__', '__eq__', '__format__', '__ge__', '__getattribute__', '__gt__', '__hash__', '__init__', '__init_subclass__', '__le__', '__lt__', '__ne__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__setattr__', '__sizeof__', '__str__', '__subclasshook__', 'edit', 'root_node', 'text', 'walk']
  1. tree.__doc__
'A Syntax Tree'
# 评价：离谱。这doc和没写有什么区别。
  1. tree.root_node。获取AST树的根节点。可以进行后续遍历操作
  2. tree.text：获取代码的字符串，二进制表示
tree.text
>>> b'def add(a, b):\n    return a+b'
  1. tree.walk(): 返回一个游走指针TreeCursor对象。这个对象拥有一些方法，可以完成树的遍历
  2. tree.edit(**params): 传入一些参数来手动修改语法树。 当源文件发生变化的时候，可以修改ast的部分参数，让翻译源文件的速度加快（来自官网，还未懂）

TreeCursor

  1. cursor = tree.walk() 返回一个TreeCursor指针
  2. cursor.__doc__
>>> 'A syntax tree cursor.'
  1. cursor.current_field_name 查不到相关资料。调用函数无任何反馈。
  2. cursor.node 获取当前指针所指向的节点 返回一个tree_sitter.Node 对象
>>> cursor.node
<Node kind=program, start_point=(0, 0), end_point=(1, 14)>
  1. cursor.goto_first_child() 指针游走到第一个孩子节点。游走成功返回True。否则返回False。游走后指针指向当前指针的第一个孩子
>>> cursor.node # 查看当前节点信息
<Node kind=ERROR, start_point=(0, 0), end_point=(0, 14)>
>>> cursor.goto_first_child() # 移动到第一个孩子
True # 移动成功，返回True
>>> cursor.node # 查看当前节点信息
<Node kind=identifier, start_point=(0, 0), end_point=(0, 3)>
>>> cursor.goto_first_child() # 移动到当前节点的第一个孩子
False # 移动失败，返回False
  1. cursor.goto_next_sibling() 移动到下一个兄弟节点
  2. cursor.goto_parent() 移动到父节点。返回bool值

tree_sitter.Node

  1. 获取方式cursor.node, tree.root_node返回的都是tree_sitter.Node对象。
  2. 所有方法
>>> root_node = tree.root_node
>>> dir(root_node)
['__class__', '__delattr__', '__dir__', '__doc__', '__eq__', '__format__', '__ge__', '__getattribute__', '__gt__', '__hash__', '__init__', '__init_subclass__', '__le__', '__lt__', '__ne__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__setattr__', '__sizeof__', '__str__', '__subclasshook__', 'child_by_field_id', 'child_by_field_name', 'child_count', 'children', 'end_byte', 'end_point', 'has_changes', 'has_error', 'is_missing', 'is_named', 'named_child_count', 'next_named_sibling', 'next_sibling', 'parent', 'prev_named_sibling', 'prev_sibling', 'sexp', 'start_byte', 'start_point', 'text', 'type', 'walk']
  1. 节点属性相关
    1. child_count 整型。子节点数量
    2. has_changes
    3. has_error 布尔型。AST的解析是否有错误
    4. is_missing
    5. is_named 是否拥有姓名？
    6. named_child_count 拥有姓名的孩子的个数
    7. end_byte 代码结束的位
    8. start_byte 代码开始的位
    9. start_point 代码开始的坐标
    10. end_point 代码结束的坐标
    11. text 该节点及子节点对应的代码字符串。
    12. type 节点类型。比如call_expression, identifier等等。
    13. sexp() 返回AST的字符串形式
  2. 遍历相关：
    1. parent 获取父节点的Node对象。
    2. children 获取所有子节点的列表
    3. next_named_sibling 获取下一个拥有姓名的兄弟节点。当不存在时，获取的时NoneType
    4. prev_named_sibling 获取前一个有名字的兄弟节点
    5. next_sibling 获取下一个兄弟节点
    6. prev_sibling 获取前一个兄弟节点
    7. walk 返回指向当前节点的指针。
    8. child_by_field_id 根据field id进行过滤。
    9. child_by_field_name 根据field name对子节点进行过滤
      1. field_name 即sexp中存在于冒号前的字符串。通常包括 name, value, left, right等。可以查看sexp具体案例展示
