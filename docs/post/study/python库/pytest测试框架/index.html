1. Pytest简介

Pytest 是用于测试 Python 应用的 Python 库。它是nose测试和unittest测试的替代方法。相比于后两者来说，Pytest更加简易。Vscode可以使用插件“测试”来自动化支持pytest。

  2. 极客教程Pytest

学完就基本掌握了。本篇剩下部分，对pytest常用功能进行总结。

3. 基本测试

在以"test_“开头或者”_test"结尾的文件中写入"test_“开头或者”_test"结尾的函数，这个函数就是测试函数，会被pytest执行。

# test_main.py

@pytest.mark.finished # 使用finished标记函数， 可以使用pytest -m finished同时测试所有被finished标记的函数
@pytest.mark.skip(reason="out-of-date") # skip会让该函数跳过
@pytest.mark.parameterize("a, b", [(1, 1), (2, 2), (3, 2)]) # 使用parameterize包含多个测试用例
@pytest.mark.parameterize("a, b", [pytest.param(1, 1, id="Test<1>"), pytest.param(2, 2, id="Test<2>"), pytest.param(3, 2, id="false")]) # 使用 pytest.param告诉这个参数的名称
def test_equal(a, b):
    # 使用断言确保程序执行。如果断言出错，则测试不成功
    assert a==b

4. 固件 fixture

固件（Fixture）是一些函数，pytest 会在执行测试函数之前（或之后）加载运行它们。 通用固件可以写在conftest.py中。在复杂的项目中，可以在不同的目录层级定义 conftest.py，其作用域为其所在的目录和子目录。注意：不要自己显式调用 conftest.py，pytest 会自动调用，可以把 conftest 当做插件来理解。

4.1. 作用域

# 不同的scope拥有不同的作用域， 可选function, module, session, class等。session是一个会话执行一次。class是一个class执行一次，需要在测试类前面写上@pytest.mark.usefixtures('class_scope')
@pytest.fixture(scope='function', name="function_rename_scope")
def func_scope():
    pass

def test_multi_scope(sess_scope, mod_scope, function_rename_scope):
    pass

4.2. 自动执行

# autouse 自动执行。不用显示调用

DATE_FORMAT = '%Y-%m-%d %H:%M:%S'
import time 

@pytest.fixture(scope='session', autouse=True)
def timer_session_scope():
    start = time.time()
    print('\nstart: {}'.format(time.strftime(DATE_FORMAT, time.localtime(start))))

    yield # yield上半部分会在程序前执行，下半部分会在程序结束后执行。

    finished = time.time()
    print('finished: {}'.format(time.strftime(DATE_FORMAT, time.localtime(finished))))
    print('Total time cost: {:.3f}s'.format(finished - start))


@pytest.fixture(autouse=True)
def timer_function_scope():
    start = time.time()
    yield
    print(' Time cost: {:.3f}s'.format(time.time() - start))

4.3. 固件参数化

# 固件参数化
@pytest.fixture(params=[
    ('redis', '6379'),
    ('elasticsearch', '9200')
]) # 给固件添加参数，可以允许获取多个参数，执行多次测试。但需要通过request.param获取
def param(request):
    return request.param

# 可以将固件参数再传给其他的函数
@pytest.fixture(autouse=True)
def db(param):
    print('\nSucceed to connect %s:%s' % param)

    yield

    print('\nSucceed to close %s:%s' % param)
